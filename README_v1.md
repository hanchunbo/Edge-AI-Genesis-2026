> **"不仅仅是程序员，更是工程架构师。"**

### 1. 技术选型权衡 (Trade-offs)

在每个季度末提交一份 **《技术决策文档》**。例如：

* 场景：分析在低算力环境下选择 **NCNN** 而非 **TensorRT** 的决策逻辑。
* 目的：训练架构师思维，掌握在有限资源下寻找最优解的能力。

### 2. 工业级代码评审 (Code Review) 标准

GitHub 协作不仅看功能，更关注以下核心指标：

* **内存零拷贝**：是否通过移动语义或引用避免了不必要的 `Memory Copy`？
* **内存对齐**：高性能计算中，循环展开与内存对齐是否优化？
* **线程安全**：并发环境下，锁的粒度是否足够细？
