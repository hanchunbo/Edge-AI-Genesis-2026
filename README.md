# Edge-AI-Genesis-2026 🚀
### —— 从硬核底层开发到 AI 边缘计算工程师的转型之路

## 📌 项目愿景
本项目由两位深耕底层开发的工程师（西工大 MEM 在读）共同发起。旨在通过 2024-2025 年的系统化学习与实践，攻克 C++ 高性能编程、模型压缩与量化、边缘端部署（NCNN/MNN/TensorRT）等核心技术，实现向 **AI 边缘计算工程师** 的职业跳槽。

## 📅 2026 转型路线图 (Roadmap)
- [ ] **Phase 1: 基础夯实 (2024 Q1-Q2)** - C++ 11/14/17 特性深度应用 & OpenCV 图像处理算法。
- [ ] **Phase 2: 模型推理 (2024 Q3-Q4)** - 深度学习基础 (CNN/Transformer) & 常用推理框架源码解析。
- [ ] **Phase 3: 边缘适配 (2025 Q1-Q2)** - 交叉编译、NPU/GPU 加速、鸿蒙/Linux 平台移植。
- [ ] **Phase 4: 实战沉淀 (2025 Q3-Q4)** - 工业级 AI 部署项目实战 & 毕业论文输出。

## 🛠 技术栈
- **Language:** C++ (Primary), Python (Scripting)
- **Frameworks:** OpenCV, NCNN, MNN, TensorRT
- **Platform:** Linux, HarmonyOS, ARM/Embedded
- **Tools:** CMake, Docker, Git, Clang-Format

## 🤝 协作守则
1. **PR 机制**：所有代码变更必须通过 Pull Request 提交，且需经另一位成员 Review 后方可合并。
2. **Issue 驱动**：每个任务（Task）先建立 Issue，讨论后再执行。
3. **性能导向**：代码需关注内存开销与耗时（Benchmark），这是边缘端开发的命脉。

- `/docs`: 存放学习笔记与行业调研报告。
- `/src/phase1_cv`: 第一阶段代码实现。
- `/benchmarks`: 性能测试数据。
